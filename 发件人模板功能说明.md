# 📧 发件人模板功能说明

## 🌟 功能概述

发件人模板功能让您可以保存常用的SMTP配置，方便在不同邮箱账号之间快速切换，无需重复输入服务器信息。

## ✨ 主要特性

1. **快速配置** - 一键应用预设的邮箱服务器配置
2. **多账号管理** - 保存多个邮箱账号的配置信息
3. **安全设计** - 密码不会被保存，确保账号安全
4. **预置模板** - 内置常见邮箱服务商的配置模板

## 📝 使用方法

### 1. 应用预置模板

系统内置了以下邮箱服务商的配置模板：

- **QQ邮箱** - smtp.qq.com, 端口465, SSL
- **163邮箱** - smtp.163.com, 端口465, SSL  
- **中国移动邮箱** - smtp.139.com, 端口465, SSL
- **Gmail** - smtp.gmail.com, 端口587, TLS

**操作步骤：**
1. 在SMTP配置页面，点击相应的邮箱标签
2. 系统自动填充服务器地址、端口和加密方式
3. 手动输入您的邮箱地址、发件人姓名和密码
4. 点击"测试连接"验证配置

### 2. 保存自定义配置

您可以保存自己的邮箱配置作为模板：

**操作步骤：**
1. 填写完整的SMTP配置信息
2. 点击"保存配置"按钮
3. 输入配置名称（如"公司邮箱"、"个人邮箱"等）
4. 点击确认保存

**注意事项：**
- 密码不会被保存，下次使用时需要重新输入
- 建议使用描述性的名称，方便识别

### 3. 管理配置模板

**删除配置：**
- 点击配置标签右上角的"×"删除自定义配置
- 预置模板（QQ、163、139、Gmail）不能删除

**更新配置：**
- 使用同名保存会覆盖原有配置

## 🔒 安全说明

1. **密码保护**
   - 系统不保存邮箱密码/授权码
   - 每次使用都需要手动输入密码
   - 保证账号安全性

2. **配置存储**
   - 配置文件保存在本地服务器
   - 仅包含服务器信息，不含敏感数据

## 💡 使用建议

### 场景1：多个邮箱账号

如果您有多个邮箱需要发送邮件：
1. 为每个邮箱创建配置模板
2. 发送时快速切换配置
3. 输入对应的密码即可

### 场景2：团队协作

团队成员可以：
1. 共享服务器配置信息
2. 各自使用自己的邮箱账号
3. 避免重复配置服务器参数

### 场景3：测试环境

开发测试时：
1. 保存测试邮箱配置
2. 保存生产邮箱配置
3. 根据需要快速切换

## 🚀 快速开始

### 中国移动邮箱配置示例

1. **开启服务**
   - 登录139邮箱网页版
   - 进入"设置" → "邮箱设置"
   - 开启"POP3/SMTP服务"

2. **获取授权信息**
   - 可以使用邮箱密码直接登录
   - 也可以生成客户端授权码

3. **应用配置**
   - 点击"中国移动邮箱"模板
   - 输入您的139邮箱地址
   - 输入密码或授权码
   - 测试连接

## ❓ 常见问题

### Q1: 为什么不保存密码？
**A:** 出于安全考虑，密码属于敏感信息，不应该以明文形式保存在配置文件中。

### Q2: 可以导出/导入配置吗？
**A:** 目前配置保存在服务器本地，暂不支持导出导入功能。

### Q3: 配置模板保存在哪里？
**A:** 配置文件保存在 `backend/templates/sender_configs.json`

### Q4: 如何添加新的邮箱服务商？
**A:** 可以通过"保存配置"功能添加任何SMTP服务器的配置。

## 📊 配置参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| 配置名称 | 用于识别的名称 | "我的QQ邮箱" |
| SMTP服务器 | 邮件服务器地址 | smtp.qq.com |
| 端口 | SMTP端口号 | 465 |
| SSL | 是否使用SSL加密 | 开启 |
| TLS | 是否使用TLS加密 | 关闭 |
| 发件人邮箱 | 您的邮箱地址 | xxx@qq.com |
| 发件人姓名 | 显示的发件人名称 | 张三 |
